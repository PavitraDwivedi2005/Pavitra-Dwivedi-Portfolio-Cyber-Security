What I Learned in My First Cybersecurity Internship

By Pavitra Dwivedi May 4, 2026 10 min read

When I first heard back from INFERA about a cybersecurity internship I was really excited and also very scared. I had studied networking played around with Nmap and done a few CTF challenges. But actually doing vulnerability assessments on real client infrastructure was a totally different thing.

This post is my story of what the experience was like what skills actually mattered and the things I wish someone had told me before the first day. If you are a computer science student thinking about getting into security I hope this helps you.

How I Got the Internship

I did not have a polished resume. I was a B.Tech student at UPES, Dehradun with an understanding of Python some experience with backend development using Django and a real interest in how systems break.

What helped me stand out I think, was being specific. Instead of just saying "cybersecurity" as a general interest I talked about the specific tools I had used. Like Wireshark for looking at packets Burp Suite for testing websites and Splunk for looking at logs. I also mentioned OWASP Top 10 vulnerabilities by name and could talk about them in the interview.

The main point is: you do not need to be an expert. You do need to show that you have actually used the tools not just read about them.

My First Week: A Reality Check

My first task at INFERA was to help do a VAPT engagement on a client's web application. I thought I would start hacking away. Instead I spent the first two days reading documents understanding the client's system and learning the team's methods.

That was my first lesson: real security work is very methodical. It is not about running tools randomly. It is about understanding what you are testing why and what the impact on the business will be.

The Tools That Actually Mattered

Here are the tools I used often during the internship:

  • Burp Suite — for intercepting and changing HTTP requests. This was the most important tool for testing web applications.
  • Nmap — for finding networks and scanning ports. This was necessary for the first part of the project.
  • OWASP ZAP — a free alternative to Burp for automatic scanning. This was great for finding easy vulnerabilities.
  • Wireshark — for looking at network traffic when we needed to understand what was happening at the packet level.
  • Splunk — for looking at SIEM logs. During incident simulations this was how we connected events across different systems.

What I Did: Finding Real Vulnerabilities

During the internship I helped find 15+ vulnerabilities in client web infrastructure. These included:

  • SQL injection in search parameters
  • Cross-site scripting in user profile fields
  • Broken access control that allowed privilege escalation
  • Insecure direct object references in API endpoints
  • Missing security headers

Finding these was not exciting. It took hours of testing each input field each API endpoint each parameter. But when you find a real vulnerability in a production system it is a great feeling.

SOC Runbooks: The Not-Exciting but Important Work

I also made 6+ SOC runbooks and incident response plans. If you do not know a runbook is basically a step-by-step guide for SOC analysts to follow when a specific alert happens.

Writing runbooks taught me something important: good security is not just about finding problems it is about building systems so teams can respond consistently. A well written runbook can mean the difference between containing an incident in minutes or hours.

My Time at IQSec Labs

At the same time I was also interning at IQSec Labs working on something different. Making data anonymous. I built a pipeline in Python that could make 80% of sensitive data anonymous while keeping 85% of its usefulness.

This might sound boring but it connected with something I care about: privacy. How do you make data useful without making it dangerous? That question led directly to my current work on scam detection that protects privacy.

Five Things I Wish I Knew Before Starting

1. Networking Basics Are Essential

I thought I understood networking from my coursework. Then I had to actually follow a request through DNS load balancers and reverse proxies. Learn TCP/IP deeply. Understand how DNS works. Know what happens when you type a URL in a browser. All of it.

2. Linux is Your Main Operating System

Almost every security tool works better on Linux. Get comfortable with the terminal. Learn bash scripting. Understand file permissions process management and networking commands.

3. Learn to Write Reports Not Just Find Bugs

A vulnerability that is not clearly documented is a vulnerability that does not get fixed. I spent as much time writing reports as I did testing. Learn to communicate risk in business terms not just technical language.

4. CTFs Are Good Practice but the Real World is Different

CTF challenges are great for building technical skills. But real-world security involves scope constraints client communication false positives and time pressure. The soft skills matter more than I expected.

5. The Community is Really Helpful

The cybersecurity community on Twitter and LinkedIn is one of the most welcoming in tech. Follow researchers read their stories and do not be afraid to ask questions. Everyone started somewhere.

What I am Doing Now

These internship experiences shaped the direction of my current work. I am building a Privacy-Preserving Real-Time Scam Detection System that uses AI to detect scams in real-time voice calls without ever storing the actual voice data. It combines everything I have learned: security architecture Python development machine learning and privacy engineering.

If you are a student considering cybersecurity my advice is simple: start building start breaking and do not wait until you feel "ready." You will learn more in your first week of real work than in months of tutorials.

Got questions? Feel free to reach out. I am happy to share what I have learned.

Frequently Asked Questions

What background do I need for a cybersecurity internship? +

A solid understanding of networking like TCP/IP DNS and HTTP along with basic Linux terminal skills and knowing at least one security tool like Burp Suite Nmap or Wireshark will put you ahead of most people applying. Knowing Python for scripting is a big plus. You do not need certifications but they can help. CompTIA Security+ is a good starting point.

How do I practise cybersecurity skills as a student? +

Start with platforms like TryHackMe which is beginner-friendly and HackTheBox which is more challenging. Set up a home lab with VirtualBox and practice with intentionally vulnerable applications like DVWA and WebGoat. Participate in CTF competitions. They are free and excellent for building practical skills.

Is a CS degree necessary for cybersecurity? +

Not strictly but it helps a lot. A CS degree gives you foundational knowledge in data structures operating systems and networking that is directly useful. That said many successful security professionals are self-taught. What matters most is demonstrable skill and hands-on experience.

What is the difference between VAPT and penetration testing? +

Vulnerability Assessment focuses on identifying and listing vulnerabilities while Penetration Testing goes further by actually trying to exploit them. In practice VAPT combines both. You scan for vulnerabilities and then test whether they can be exploited giving a complete picture of the security posture.
